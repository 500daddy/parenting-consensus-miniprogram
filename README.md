# parenting-consensus-miniprogram

# 养娃新手村微信小程序 MVP

这是一个原生微信小程序 MVP，核心闭环是：搜索育儿问题后，展示主流共识、观点比例、权威来源、少数观点和安全提醒。当前版本全部使用本地 mock 数据，不依赖后端或 npm。品牌名为“养娃新手村”，slogan 为“养娃路上，先来村里问问”。

## 如何运行

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择 `C:\Users\zc\Documents\New project\parenting-consensus-miniprogram`。
4. 当前 `project.config.json` 已配置 AppID；如需在其他账号调试，可在微信开发者工具里切换为自己的测试号。

## 已完成

- 首页：新手村 hero、搜索入口、分类 chip、村里大家都在问、今日村里共识、安全红线。
- 问答搜索：推荐问题、历史记录、关键词跳转结果页。
- 问题汇总：主流共识、观点比例、理由卡片、权威内容、少数观点、安全提醒、收藏。
- 权威观点：来源类型筛选、可信提示、注意事项、相关问题。
- 收藏：底部 tab 一级入口，便于回看常查共识问题。
- 分类：按育儿场景筛选问题，当前作为首页分类 chip 的二级入口。
- 我的：登录入口、宝宝档案编辑、历史、反馈占位、免责声明。
- 结果页：主流共识、观点比例、权威内容、安全提醒、前置健康内容边界和高风险提示。
- 上线准备：发布态配置已开启请求域名校验和代码压缩；当前版本仍保留 source map，方便小范围种子用户阶段排查问题。
- 当前热门 6 个问题均已补齐完整 mock 共识结果，可从首页、搜索、分类、收藏、历史等入口闭环查看。
- 首页 hero 使用本地新手村家庭插画资源：`miniprogram/assets/hero/village-hero.png`。

## 验证建议

- 运行 `node scripts/sanity-check.js`，检查页面文件、JSON、JS 模块、tab 图标、mock 数据关联、观点比例和常见关键词路由是否完整。
- 发布前按 `docs/release-checklist.md` 做微信开发者工具预览、核心路径验收和小程序后台资料检查。
- GitHub Actions 已配置同一条 sanity check，推送到 `main` 或创建 PR 时会自动执行。
- 搜索“宝宝发烧能洗澡吗”，检查结果页内容是否完整。
- 分别搜索“宝宝咳嗽有痰”“挑食不爱吃蔬菜”“疫苗后低烧”，检查是否进入对应结果页。
- 在结果页点击收藏，再到底部“收藏”tab 查看。
- 在首页点击分类 chip，确认进入分类页后筛选正确。
- 在“我的”页点击登录，登录后编辑宝宝昵称、月龄、性别、过敏史并保存。
- 退出登录后，确认宝宝档案回到未登录空状态，收藏和历史仍保留。
- 在“我的”页查看内测反馈、隐私说明、审核备注和免责声明弹窗。
- 抽查发热、用药、疫苗、皮肤过敏、安全急救类结果页，确认页面前部展示健康内容边界或高风险提醒。
- 在权威观点页切换“儿科医生 / 医学指南 / 育儿百科 / 认证达人”。
- 检查底部 tab 的选中态是否明显，页面内容是否被底栏遮挡。
