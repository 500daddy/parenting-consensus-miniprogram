# 待补充问题后台规范

日期：2026-05-06

## 目标

上线后把“没有搜到精确共识”的问题集中回收，形成可审核、可去重、可转入正式题库的增长闭环。

当前小程序版本只保存到本机缓存；正式接后端后，提交按钮应写入后台集合或数据表。

## 提交前校验

- 问题长度：6 到 40 个字。
- 不能只提交泛词，例如“发烧”“分床”“咳嗽”“辅食”。
- 需要包含育儿相关词，例如“宝宝”“孩子”“月龄”“发烧”“辅食”“夜醒”“湿疹”等。
- 需要像一个问题，例如包含“怎么办”“正常吗”“要不要”“能不能”“多久”“为什么”等表达。
- 页面提示示例：`宝宝夜里频繁醒怎么办？`

## 建议字段

- `id`：后台生成的唯一 ID。
- `rawQuestion`：用户原始提交文本。
- `normalizedQuestion`：去空格、统一标点后的问题。
- `sourcePage`：提交来源，例如 `search_empty`、`result_empty`。
- `searchKeyword`：触发未命中的搜索词。
- `matchedQuestionId`：如果曾经命中过相近问题，记录候选题 ID。
- `clickedRecommendationId`：无结果页推荐问题被点击时记录，方便判断是否真的解决。
- `categoryGuess`：后台或规则初判分类，例如发烧护理、睡眠、辅食、皮肤过敏。
- `babyAgeRange`：可选，来自用户档案或提交时选择，注意不强制填写。
- `anonymousId`：匿名设备或用户标识，用于去重和频次统计。
- `userOpenId`：仅在用户授权并接入登录后记录。
- `hitCount`：相同或高度相似问题的累计次数。
- `status`：处理状态。
- `reviewerNote`：人工审核备注。
- `promotedQuestionId`：转入正式题库后的问题 ID。
- `createdAt`：首次提交时间。
- `updatedAt`：最近更新时间。

## 状态流转

- `pending`：已提交，待初筛。
- `triaged`：已归类，确认有整理价值。
- `drafting`：正在补充答案和参考来源。
- `reviewed`：内容已审核，可进入题库。
- `published`：已上线到正式问题库。
- `ignored`：过短、重复、非育儿问题、无法整理或不适合回答。

## 去重规则

- 先按 `normalizedQuestion` 精确去重，相同问题只增加 `hitCount`。
- 再按关键词和语义相似聚类，例如“宝宝半夜老醒怎么办”和“宝宝夜醒频繁怎么办”归为同一簇。
- 每周优先整理 `hitCount` 高、风险高、朋友测试中反复出现的问题。

## 隐私与边界

- 不强制收集宝宝姓名、生日、手机号、微信昵称等个人信息。
- 需要用户登录时，只用于跨设备同步和问题提交统计，不用于分享文案中拼接微信昵称。
- 医疗、用药、急救类问题必须保留健康提醒，不能输出诊断、处方或个体化用药结论。

## 运营节奏

- 每周导出新增问题，按分类和热度排序。
- 先补 P0：发烧、用药、咳嗽、皮肤过敏、疫苗反应、误食噎住。
- 每次上线题库更新后，回看待补充池中是否还有高频未覆盖问题。
