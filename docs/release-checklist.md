# 发布与验收清单

- [ ] 使用微信开发者工具导入当前仓库根目录，例如本机为 `C:\Users\admin\Documents\New project`。
- [ ] 确认首页、问问、收藏、我的四个底部 tab 选中态明显。
- [ ] 检查页面底部内容没有被自定义 tab bar 遮挡。
- [ ] 重点预览新分类 icon，尤其“皮肤过敏”和“安全急救”的像素图标观感。
- [x] 静态校验已覆盖分类、问题、权威来源和 tab 图标路径，当前没有兜底问号图标暴露。
- [x] 运行 `node scripts/sanity-check.js`，确认输出 `project sanity ok`。

## 核心路径验收

- [ ] 搜索“宝宝发烧能洗澡吗”，确认进入完整结果页。
- [ ] 搜索“宝宝咳嗽有痰”“挑食不爱吃蔬菜”“疫苗后低烧”，确认路由到对应结果页。
- [ ] 在结果页点击收藏，再到底部“收藏”tab 查看。
- [ ] 从首页点击分类 chip，确认进入分类页后筛选正确。
- [ ] 首页“大家在问”底部点击“查看更多问题”，确认进入问问页。
- [ ] 结果页“可用小工具”只显示推荐入口，点开后可选择合适工具；推荐过多时不直接堆在结果页正文里。
- [ ] 结果页工具推荐应和当前问题语义相关，例如身高体重问题优先推荐生长记录，不展示无关的疫苗记录。
- [ ] 结果页工具弹层中可跳转“小工具与记录”，让用户查看全部可用工具。
- [ ] 结果页底部“相关问题”只显示两张卡片，右上“更多问题”可跳转问问页。
- [ ] 在“我的”页登录后编辑宝宝昵称、月龄、性别、过敏史并保存。
- [ ] 保存宝宝档案后进入任一结果页，确认展示“按宝宝档案提醒”，且文案仍保持参考边界。
- [ ] 退出登录后，确认宝宝档案回到未登录空状态，收藏和历史仍保留。
- [ ] 查看反馈说明、隐私说明和免责声明弹窗；“审核备注”不在用户端展示。
- [ ] 在“帮我们补问题”里可以提交新的育儿问题；有历史问题时可以选择继续提交或查看已提交，且文案明确只保存在本机。
- [ ] 抽查发热、用药、疫苗、皮肤过敏、安全急救类结果页，确认前部展示健康内容边界或高风险提醒。
- [ ] 在权威观点页切换“儿科医生 / 医学指南 / 育儿百科 / 认证达人”。

## 小工具与记录验收

- [ ] 从“我的”页进入“小工具与记录”，空状态只显示“还没有记录”和“新建记录”，不显示四个工具卡片。
- [ ] 有记录后进入“小工具与记录”，首屏直接展示记录筛选和记录列表；右上角“新建”可选择就医、奶量、疫苗、生长四类记录。
- [ ] 新建奶量记录时，未填写宝宝信息会显示“去完善宝宝信息”；保存宝宝信息后可返回奶量记录页，已输入的奶量、时间、备注不丢失，宝宝和月龄自动带入。
- [ ] 新建就医前记录、疫苗记录、生长记录时，也能通过“去完善宝宝信息”进入独立宝宝信息页，保存后回到原工具页。
- [ ] 宝宝信息页的月龄和性别滚动选择中不出现“未设置”；“未设置”只作为未填写时的展示文案。
- [ ] 奶量记录保存后，记录页展示每日奶量汇总；日期超过 2 天时可以展开/收起。
- [ ] 奶量记录显示当日次数和总量；多条奶量、疫苗、生长记录默认只展示最新一条，可展开全部，再收起。
- [ ] 疫苗记录选择“其他疫苗”或“其他反应”时展示自定义输入框，未选择其他时不展示；疫苗厂商为选填项。
- [ ] 生长记录页优先展示记录输入区，国内标准参考卡片放在下方；有性别和月龄时显示本月龄参考范围和与参考中位数差异。
- [ ] 生长记录页缺少宝宝性别或月龄时，提示去完善宝宝信息，并能跳转独立宝宝信息页后返回当前工具页。
- [ ] 就医记录保存后，症状、已用过什么和备注都应在记录详情卡片中显示；标题使用工具标题“就医前记录单”，不是问题本身。
- [ ] 记录列表按“全部 / 就医 / 喂养 / 疫苗 / 生长”筛选时，各类型记录有清晰标签区分，不依赖背景边框颜色识别。
- [ ] 编辑已有记录后再次保存，应覆盖原记录，不新增重复记录。
- [ ] 删除任一记录后，列表和汇总数据即时更新；删除全部后回到空状态。
- [ ] 搜索历史列表不显示不匹配的重复图标，点击历史项可进入对应结果页。
